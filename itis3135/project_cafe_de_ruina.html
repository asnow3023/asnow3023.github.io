Project Home Page

Project Design: Cafe de Ruina


Project Overview

  • The project will be about creation of website with interactive visual novel application. The website will be
    about backgrounds of Library of Ruina characters to satisfy fans of the gamewith a organized character
    settings and additional side stories for each characters.
  • The primary target userbase of this application is previous users who played the game before, as well as
    other potential players who may be interested to play after using the application.
  • The application will guide users through different webpages with each different characters from the game:
    “Library of Ruina.” Users will be able to view organized background stories, character settings, and interact
    with characters with pre-scripted side storyline for additional entertainment.

Page Design

Home Page

  1. Purpose: The purpose of the page is to guide users through all other pages and appeal the viewers to explore the website with clean design.
  2. Audience/Users: Fans of Type Moon games and Type Moon associates.
  3. What will be content of the page? The page will contain navigations and initial guidelines to explain how navigation bar works.
  4. Is this a page where you are asking users to enter data on the page?Y es, users be able to enter their name through a prompt and proceed through the story with the input name.
  5. Will those data fields need any validations (if any)? Yes, the name cannot be blank. Name can be numbers, words, or special characters (corresponds to Library of Ruina settings)
  6. Will the page contain buttons or hyperlinks or drop downs? The page will contain buttons that takes the user to page for each corresponding characters.
  7. What actions will happen in the page? User will be able to select desired button for navigation to all other webpages.


Contact us

  1. Purpose: The purpose of this page is intended for users to contact with the site developer for any inquiries or design requests.
  2. Audience/Users: All viewers of the site / fans of Library of the Ruina / Project Moon associate
  3. What will be content of the page? Primary contact information for the design firm and user inputs for inquiries.
  4. Is this a page where you are asking users to enter data on the page? Yes, users will enter their contact information and messages to us.
  5. Will those data fields need any validations (if any)? No, all messages will be received.
  6. Will the page contain buttons or hyperlinks or drop downs? The page will contain user input labels and submit button.
  7. What actions will happen in the page? User input their information and desired messages to developers in the given space. User will then submit their inquiries if needed.


About us

  1. Purpose: The purpose of this page is intended for users view information of about design firm, history, and the mission statement of the firm.
  2. Audience/Users: All viewers of the site with interest to request works to the firm. Project Moon associates
  3. What will be content of the page? List of members in the design firm, mission statement, and explanation on why I chose to create this website.
  4. Is this a page where you are asking users to enter data on the page? No, users will only be able to view the page and its contents.
  5. Will those data fields need any validations (if any)? No, the validation is not needed.
  6. Will the page contain buttons or hyperlinks or drop downs? No, the page will not contain hyperlinks or dropdowns.
  7. What actions will happen in the page? User will get to view the information about staff member and design firm responsible for creating the website for their own purposes.


Story

  1. Purpose: The purpose of this page is intended for users to understand the background on current timeline in the story and settings.
  2. Audience/Users: Viewers & fans interested in fan-made story of Library of Ruina.
  3. What will be content of the page? A walkthrough on summary of story so far and explanation of background on the world of Library of Ruina.
    The walkthrough will be displayed in a form of book with pages being able to flip over through mouse left and right clicks.
  4. Is this a page where you are asking users to enter data on the page? No, users will interact only with mouse clicks.
  5. Will those data fields need any validations (if any)? No, the data fields will not need any validations.
  6. Will the page contain buttons or hyperlinks or drop downs? No, the page will only have on click events.
  7. What actions will happen in the page? User gets to right and left click through given page to flip over and view information on each page of the book.

Dynamic Functionality

The functionality I would like to utilize on my websites are interactive images and photo gallery with animated motions.
The interactive images would be implemented for most of the pages to create a story event for each character.
Photo gallery would be used to show in-game scenes of that character during the data and the ending page of the story (Angela’s Hall).
